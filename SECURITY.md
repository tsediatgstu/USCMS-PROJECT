
# Security Policy

## Supported Versions

We are committed to the security of our users. We currently provide security updates for the following versions:

| Version | Supported | Notes |
| --- | --- | --- |
| **5.1.x** | ✅ | **Current Stable:** Receives all bug fixes and security patches. |
| 5.0.x | ❌ | **End of Life:** No longer receiving updates. Please upgrade. |
| **4.0.x** | ✅ | **LTS Support:** Receives critical security patches only. |
| < 4.0 | ❌ | **Unsupported:** Retired versions. |

## Reporting a Vulnerability

We value the work of the security community and appreciate responsible disclosure. If you discover a vulnerability, please follow the process below to ensure the issue is resolved safely.

### 🛡️ How to Submit a Report

**Please do not open a public issue for security vulnerabilities.** The preferred method for reporting is via **GitHub Private Vulnerability Reporting**. This allows us to discuss and fix the issue in a secure environment without alerting malicious actors.

* Navigate to the **Security** tab of this repository.
* Click on **Advisories** on the left sidebar.
* Click **Report a vulnerability**.

Alternatively, you may email our security team at **security@example.com** using the subject line `[Vulnerability Report] <Description>`.

### 📝 What to Include

To help us triage your report, please include:

* **A Detailed Description:** The nature of the vulnerability and potential impact.
* **Steps to Reproduce:** A clear Proof of Concept (PoC) script, screenshots, or step-by-step instructions.
* **Affected Components:** Specific files, functions, or API endpoints involved.

### 🕒 Our Response Timeline

We aim to maintain an open line of communication with researchers:

1. **Acknowledgement:** Within **48 hours** of submission.
2. **Triage & Validation:** Within **5 business days**, at which point we will confirm if the report is accepted or declined.
3. **Status Updates:** Every **7 days** until the vulnerability is resolved.
4. **Public Disclosure:** Once a patch is released, we will publish a GitHub Security Advisory and credit you for your discovery (unless you prefer to remain anonymous).

### ⚖️ Responsible Disclosure Guidelines

We ask that you:

* Avoid data deletion, service disruptions, or privacy violations.
* Give us a reasonable timeframe to remediate the issue before making any information public.
* Do not engage in extortion or demand payment in exchange for the report.
